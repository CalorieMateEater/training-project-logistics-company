## 4. 運用方針
- メンバー異動、権限変更、兼務変更が発生した場合は当日中に管理簿を更新する。
- 役割分担変更時は体制図とバックアップ体制表を同時更新する。
- 月初に参画状況と権限棚卸を実施する。
- 要件の意味を変更した場合は、新しい整合性確認サイクルを開始し、トレーサビリティ表とチェックリストの状態を空欄へ戻してから再確認する。
- 詳細設計の処理内容、データ項目、IF、例外条件、状態遷移、DB更新仕様を変更した場合は、設計実装整合性管理の確認サイクルを更新し、該当行の状態を空欄へ戻してからJava実装との差分を確認する。
