## 4. 更新ルール
- 確認した時点で状態、最終確認日、確認者を更新する。
- `実装乖離あり`または`設計確認要`の場合は課題IDを必ず記載する。
- 確認サイクルを更新した場合は、該当行の状態を空欄へ戻し、進捗集計を再計算する。
- Java修正を行った場合は、確認完了前に `.\gradlew.bat spotlessApply` と `.\gradlew.bat build` を実行する。
