# 設計実装整合性管理概要

## 1. 目的
詳細設計を正本として、Java実装へ設計内容が漏れなく反映されていることを継続的に確認する。
設計変更後に古い確認結果を流用しないよう、確認サイクル、チェック状態、課題を管理する。

## 2. 管理資料
- [設計実装トレーサビリティ表](./設計実装トレーサビリティ表.md)
- [実装整合性確認チェックリスト](./実装整合性確認チェックリスト.md)
- [実装乖離・課題管理簿](./実装乖離・課題管理簿.md)

## 3. 対象範囲
| 区分 | 対象 |
| --- | --- |
| 設計正本 | `docs/自社内部設計/業務設計/処理設計書`、`処理機能設計書`、`メソッド設計書`、`クラス設計書`、`データモデル設計書` |
| 実装対象 | `java/hoge-orderhub-common`、`java/hoge-orderhub-batch`、`java/hoge-orderhub-worker`、`java/hoge-shipping-gateway-api`、`java/hoge-stock-keeper-api`、`java/hoge-customer-registry-api` |
| 確認単位 | 処理設計書ID、処理機能ID、主要Controller/Service/Mapper/Repository/Entity/DTO |
| 対象外 | README、PDF、設計書の表記ゆれ、未実装を前提にした将来拡張メモ |

## 4. 状態定義
| 状態 | 意味 |
| --- | --- |
| 空欄 | 現在の確認サイクルでは未確認 |
| `確認中` | 設計と実装を確認中で、結果が確定していない |
| `整合` | 詳細設計とJava実装に明確な乖離がない |
| `実装乖離あり` | 詳細設計に対してJava実装が不足、過剰、または挙動不一致 |
| `設計確認要` | 実装差分ではなく、詳細設計の記載不足または曖昧さが原因で判断できない |
| `実装対象外` | 現時点ではJava実装対象外であることが明記されている |

## 5. 確認サイクル
確認サイクルIDは `IMPL-CYC-yyyyMMdd-NN` 形式で採番する。

| 項目 | 現在値 |
| --- | --- |
| 現在の確認サイクル | `IMPL-CYC-20260628-01` |
| 基準日 | 2026-06-28 |
| 確認対象 | 詳細設計とJava実装 |
| 状態 | 初回確認準備 |

## 6. 設計更新時のリセット手順
詳細設計の処理内容、データ項目、IF、例外条件、状態遷移、DB更新仕様が変わった場合は、Java修正前に次を実施する。

1. 新しい確認サイクルIDを採番する。
2. [設計実装トレーサビリティ表](./設計実装トレーサビリティ表.md)の該当行の状態、最終確認日、確認者を空欄へ戻す。
3. [実装整合性確認チェックリスト](./実装整合性確認チェックリスト.md)の該当行の状態、最終確認日、確認者を空欄へ戻す。
4. 未解決課題がある場合は[実装乖離・課題管理簿](./実装乖離・課題管理簿.md)へ登録する。
5. 詳細設計を確認してからJava実装を修正する。
6. Java修正後は `java` ディレクトリで `.\gradlew.bat spotlessApply` と `.\gradlew.bat build` を実行する。
7. 確認結果をトレーサビリティ表とチェックリストへ反映する。

誤字、リンク修正、意味を変えない表記統一のみの場合は、確認状態をリセットしない。

## 7. 確認観点
- 処理設計書に記載された入力、出力、正常系、異常系、冪等性、リトライ、状態更新が実装されていること。
- メソッド設計書に記載されたバリデーション、DB CRUD、外部通信、メッセージ削除条件が実装されていること。
- クラス設計書に記載された責務がController、Service、Mapper、Repository、DTO、Entityへ過不足なく分離されていること。
- データモデル設計書のテーブル、カラム、主キー、ユニーク制約、履歴保持がEntityとRepositoryに反映されていること。
- テストが処理単位の主要な正常系、異常系、境界値、冪等性を確認していること。

## 8. Codexへの依頼例
```text
設計実装整合性管理資料の現在の確認サイクルを確認してください。
実装整合性確認チェックリストの未確認項目から、StockKeeper関連を確認してください。
詳細設計を正本として、明確な乖離がある場合だけJavaを修正してください。
設計確認要のものは推測で実装せず、実装乖離・課題管理簿へ登録してください。
確認後、設計実装トレーサビリティ表とチェックリストを更新してください。
```
