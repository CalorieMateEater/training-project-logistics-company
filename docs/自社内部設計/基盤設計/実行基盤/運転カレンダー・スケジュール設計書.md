# 運転カレンダー・スケジュール設計書

## 1. 目的
本書は、Hoge社の出荷・配送管理基盤における24時間運転対象、営業時間制約対象、定期実行ジョブ、日次運用タイミングを定義する。

## 2. 基本方針
- Hoge社の受付系処理は24時間365日で稼働させる。
- Bar社向け出荷依頼送信のみ、Bar社営業時間の制約を受ける。
- 日次運用処理は業務本線の負荷が低い時間帯に集約する。

## 3. 稼働区分
| 区分 | 対象 | 稼働条件 |
| --- | --- | --- |
| 常時稼働 | 出荷依頼受付API、出荷状態照会API、配送会社連携Worker、配送結果受付API、配送状態取込Worker、配送結果返却Worker | 24時間365日 |
| イベント起動 | FooOrderImportJob | Foo社HULFT受信完了都度 |
| 定期起動 | HULFT送信ポーリングジョブ | 5分間隔 |
| 定期起動 | 日次アーカイブBatch | 毎日03:30 JST |
| 時間制約あり | Bar社向け出荷依頼送信 | 平日08:00-18:00 JST |

## 4. 日次スケジュール
| 時刻 | 処理 | 備考 |
| --- | --- | --- |
| 00:00-24:00 | Foo注文受付、Hoge直受注登録、状態照会 | 常時受付 |
| 毎時00分-59分 | Bar送信待ちキュー監視 / Fuga送信待ちキュー監視 | Barは営業時間外待機、Fugaは即時送信滞留確認 |
| 08:00-18:00 平日 | Bar出荷依頼送信 | 営業時間内のみ送信実施 |
| 03:30 | 日次アーカイブBatch 起動 | 前日分対象 |
| 03:30-04:30 | アーカイブ出力、保管 | 完了目安60分以内 |
| 5分間隔 | HULFT送信ポーリング | Foo向け受付通知、配送結果返却送信 |

## 5. Bar営業時間カレンダー
| 項目 | 内容 |
| --- | --- |
| 基本営業時間 | 平日 08:00-18:00 JST |
| 営業日 | 月曜日から金曜日 |
| 休業日 | 土日、Bar社指定休業日 |
| Hoge側扱い | Bar休業日は営業時間外と同じ扱いで送信待機 |

## 6. 障害・保守時の扱い
| 事象 | 方針 |
| --- | --- |
| Hoge社夜間保守 | 状態照会API、受付APIの停止有無を事前告知し、Bar/Fuga送信待ちキューは停止期間中保持する |
| Bar社計画停止 | 停止期間中は送信待ち継続とし、復旧後の営業時間から順次送信する |
| HULFT停止 | 受信済ファイルはS3保管を正本とし、復旧後に再送再開する |

## 7. 監視観点
| 観点 | 監視内容 |
| --- | --- |
| 営業時間内未送信滞留 | `bar-shipment-request-queue.fifo` 滞留件数、`fuga-shipment-request-queue.fifo` 即時送信滞留件数 |
| 夜間受付継続 | Foo受信件数、Hoge直受注登録件数 |
| 日次ジョブ | アーカイブBatch開始・終了、異常終了有無 |
| HULFT返却 | 未送信ファイル件数、送信失敗件数 |
