# ジョブネット設計書

## 1. 目的
OrderHub のイベント駆動ジョブ、定期実行ジョブ、および常駐 Worker の起動契機、前後関係、再実行条件を整理する。

## 2. ジョブネット一覧
| ジョブネット ID | 名称 | 起動契機 |
| --- | --- | --- |
| `JN-HOGE-001` | Foo 注文取込ジョブネット | HULFT受信完了イベント |
| `JN-HOGE-002` | 配送会社送信制御ジョブネット | `bar-shipment-request-queue.fifo` または `fuga-shipment-request-queue.fifo` 着信 |
| `JN-HOGE-003` | 配送結果反映ジョブネット | Bar社またはFuga社通知受信 |
| `JN-HOGE-004` | Foo 結果返却ジョブネット | 配送結果反映後 |
| `JN-HOGE-005` | 日次アーカイブジョブネット | 日次定刻 |

## 8. 再実行ルール
| ジョブ | 再実行可否 | 条件 |
| --- | --- | --- |
| FooOrderImportJob | 可 | 同一ファイル未完了時のみ。完了済みは不可 |
| 配送会社 API 送信 | 可 | 同一 `shipment_request_id` は冪等キーで再送 |
| 配送状態取込Worker | 可 | 同一 `order_id + status_seq` は冪等更新 |
| 返却ファイル生成 | 可 | 未送信状態の場合のみ |
| 日次アーカイブBatch | 可 | 同一 `archive_execution_id` 未完了時のみ |

## 9. 運用上の注意
- ジョブネット単位で `run_id` を払い出し、ログと連携履歴に記録する。
- `JOB-HOGE-002` 失敗時は `JOB-HOGE-003` を起動しない。
- `EVT-HOGE-104` が継続する場合でも異常終了ではなく、営業時間待機として扱う。
- HULFT 送信失敗時は `JOB-HOGE-303` の再送判定を行い、3回超過で手動対応へ移行する。
