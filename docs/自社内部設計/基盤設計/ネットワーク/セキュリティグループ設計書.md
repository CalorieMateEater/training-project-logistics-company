# セキュリティグループ設計書

## 1. 目的
Hoge社 AWS 環境における Security Group の適用先と、インバウンド/アウトバウンド許可条件を定義する。

## 2. 設計方針
- 通信許可は Security Group 参照を優先する。
- Batch/Worker は原則インバウンドなしとする。
- `0.0.0.0/0` 宛アウトバウンドは設定しない。
- 社外通信は VPN 配下の対向 CIDR に限定する。

## 3. Security Group 一覧
| SG 名 | 適用先 | 役割 |
| --- | --- | --- |
| `sg-hulft-rx` | HULFT受信サーバ | Foo社 HULFT 受信 |
| `sg-hulft-tx` | HULFT送信サーバ | Foo社 HULFT 送信 |
| `sg-external-api-alb` | Shipping API 用 ALB | Foo状態照会、Hoge直受注入口 |
| `sg-carrier-callback-alb` | 配送結果受付 ALB | Bar/Fuga 結果通知入口 |
| `sg-internal-api-alb` | Internal API Gateway | 社内 API 入口 |
| `sg-orderhub-batch` | FooOrderImportJob | 取込 Batch |
| `sg-bar-worker` | 配送会社連携Worker | Bar/Fuga 宛送信 |
| `sg-result-worker` | 配送状態取込Worker | Bar/Fuga 結果取込 |
| `sg-notify-worker` | 配送結果返却Worker | Foo 向け返却ファイル生成 |
| `sg-archive-batch` | 日次アーカイブBatch | アーカイブ処理 |
| `sg-shipping-api` | 出荷依頼受付 API / 出荷状態照会 API | API 処理 |
| `sg-customer-registry` | Customer Registry | 顧客確認 |
| `sg-stock-keeper` | Stock Keeper | 在庫管理 |
| `sg-orderhub-db` | RDS | OrderHub データストア |
| `sg-vpce-common` | Interface Endpoint | 共通 AWS サービス接続 |

## 4. インバウンド / アウトバウンド
| SG 名 | Inbound | Outbound |
| --- | --- | --- |
| `sg-hulft-rx` | Foo社 CIDR から `TCP/30000` | S3 / SSM Endpoint `TCP/443` |
| `sg-hulft-tx` | なし | Foo社 CIDR `TCP/30000`、S3 / SSM Endpoint `TCP/443` |
| `sg-external-api-alb` | Foo社 CIDR `TCP/443`、Hoge社業務NW CIDR `TCP/443` | `sg-shipping-api` `TCP/8443` |
| `sg-carrier-callback-alb` | Bar社 CIDR `TCP/443`、Fuga社 CIDR `TCP/443` | `sg-result-worker` `TCP/8443` |
| `sg-internal-api-alb` | `sg-orderhub-batch` `TCP/443`、Hoge社倉庫NW CIDR群 `TCP/443` | `sg-customer-registry` `TCP/8443`、`sg-stock-keeper` `TCP/8443` |
| `sg-orderhub-batch` | なし | Internal API `443`、RDS `5432`、S3/SQS/Secrets/Logs `443` |

| `sg-notify-worker` | なし | RDS `5432`、S3/Secrets/Logs `443` |
| `sg-archive-batch` | なし | RDS `5432`、S3/KMS/Logs `443` |
| `sg-shipping-api` | `sg-external-api-alb` `8443` | RDS `5432`、SQS/Secrets/Logs `443` |
| `sg-customer-registry` | `sg-internal-api-alb` `8443` | Secrets/Logs `443` |
| `sg-stock-keeper` | `sg-internal-api-alb` `8443` | Secrets/Logs `443` |
| `sg-orderhub-db` | App/API 系 SG から `5432` | なし |

## 5. 採用ポート
| ポート | 用途 |
| --- | --- |
| `443/TCP` | HTTPS、AWS API |
| `8443/TCP` | ECS サービス実ポート |
| `5432/TCP` | PostgreSQL |
| `30000/TCP` | HULFT |

## 6. 補足
- ECS タスクは `awsvpc` モードを利用し、タスク単位に Security Group を適用する。
- EC2 への SSH は許可しない。運用保守は Session Manager を利用する。
- 倉庫担当者端末は AWS 内リソースを直接参照せず、倉庫NW から `sg-internal-api-alb` 配下の Internal API Gateway にのみ到達可能とする。
