# データストア設計書

## 1. 目的
OrderHub データストアおよび S3 バケット群の構成、接続、バックアップ、保持期間、暗号化方針を定義する。

## 2. データストア一覧
| 種別 | サービス | 用途 |
| --- | --- | --- |
| RDB | Amazon RDS for PostgreSQL | 注文、配送状態、連携履歴 |
| オブジェクト | Amazon S3 | 着信、処理済み、返却、アーカイブ |

## 3. RDS 設計
| 項目 | 内容 |
| --- | --- |
| エンジン | PostgreSQL 15 |
| 構成 | Multi-AZ |
| 接続方式 | RDS Proxy 経由 |
| インスタンスクラス | `db.m6g.large` |
| ストレージ | gp3 300GB |
| 保管時暗号化 | KMS |
| 転送時暗号化 | TLS 必須 |

## 4. RDS 接続方針
| 項目 | 内容 |
| --- | --- |
| 最大アプリ接続 | 200 |
| 接続プール | RDS Proxy を利用 |
| 接続タイムアウト | 5秒 |
| クエリタイムアウト | 30秒 |

## 5. バックアップ / 復旧
| 項目 | 内容 |
| --- | --- |
| 自動バックアップ保持 | 35日 |
| PITR | 有効 |
| 手動スナップショット | 月次 12世代 |
| 復旧訓練 | 四半期ごと |
| 目標復旧 | RTO 30分、RPO 5分 |

## 6. S3 バケット設計
| バケット | 用途 | Versioning | ライフサイクル |
| --- | --- | --- | --- |
| `hoge-orderhub-landing-prod` | Foo 着信ファイル | 有効 | 30日後に削除 |
| `hoge-orderhub-processed-prod` | 処理済みファイル | 有効 | 90日保持 |
| `hoge-orderhub-status-prod` | Foo 返却ファイル | 有効 | 30日保持 |
| `hoge-orderhub-archive-prod` | 売上配送実績アーカイブ | 有効 | 365日後に IA、7年保持 |

## 7. データ保持
| データ | 保持期間 |
| --- | --- |
| 注文・配送状態 | 3年 |
| 連携履歴 | 1年 |
| 監査ログ | 7年 |
| アーカイブファイル | 7年 |

## 8. 性能方針
- OrderHub テーブルは `order_id`、`shipment_request_id`、`partner_order_id`、`status_updated_at` に索引を付与する。
- 配送状態更新は `order_id + status_seq` で一意制御する。
- アーカイブ出力は夜間帯に限定し、業務本線と IO 競合しないようにする。
