# 在庫管理基本設計書

## 1. 目的
本書は、Hoge Stock Keeper における在庫数量モデル、倉庫場所管理、在庫ライフサイクル、排他制御、主要テーブルを定義し、OrderHub から参照される在庫管理の基本設計を整理する。

## 2. 設計方針
- 在庫は `商品コード + 倉庫場所コード` 単位で管理する。
- 数量は `保有在庫数` と `引当済在庫数` を分けて保持し、`利用可能在庫数 = 保有在庫数 - 引当済在庫数` とする。
- 引当成功時は保有在庫数を減算せず、引当済在庫数のみを加算する。
- 引当解除時は引当済在庫数のみを減算する。
- 出荷確定時は保有在庫数と引当済在庫数を同時に減算する。
- 将来の配送会社選定に備え、引当結果には倉庫場所コードを含めて OrderHub へ返却する。

## 3. 在庫ライフサイクル

### 3.1 数量定義
| 項目 | 説明 |
| --- | --- |
| 保有在庫数 | Hoge社が対象倉庫場所に物理保有している数量 |
| 引当済在庫数 | 受注により予約済で、まだ出荷確定していない数量 |
| 利用可能在庫数 | `保有在庫数 - 引当済在庫数` |

### 3.2 業務イベントと数量変化
| イベント | 保有在庫数 | 引当済在庫数 | 備考 |
| --- | --- | --- | --- |
| 在庫引当 | 変化なし | 増加 | 倉庫場所を確定する |
| 在庫引当解除 | 変化なし | 減少 | 未出荷取消、障害復旧時 |
| 在庫出荷確定 | 減少 | 減少 | 初版では `配送受付済` を契機とする |

## 4. 倉庫場所設計
| 項目 | 内容 |
| --- | --- |
| 管理単位 | `warehouse_location_code` |
| 想定用途 | 在庫引当先の保持、将来の配送会社選定、配送制約判定、拠点別在庫分析 |
| 初版での扱い | 1注文明細につき1倉庫場所を返却する |
| OrderHub への連携 | 在庫引当結果と出荷依頼に倉庫場所コードを保持する |

## 5. API設計要点
| IF-ID | API | 主処理 |
| --- | --- | --- |
| `IF-HOGE-STK-001` | 在庫引当API | 利用可能在庫を確認し、倉庫場所を確定して引当済在庫数を更新 |
| `IF-HOGE-STK-002` | 在庫引当解除API | 指定引当を解除し、引当済在庫数を戻す |
| `IF-HOGE-STK-003` | 在庫出荷確定API | 指定引当を出荷確定し、保有在庫数と引当済在庫数へ反映 |

## 6. 排他制御方針
- 排他制御単位は `商品コード + 倉庫場所コード` とする。
- 在庫残数更新時は在庫残高テーブルに対して排他ロックを取得する。
- 実装方式は行ロック、または `version_no` による楽観ロックを許容するが、初版基本設計では `version_no` を保持する。
- 排他待ちタイムアウト時は再送可能異常として OrderHub へ返却する。

## 7. テーブル設計

### 7.1 倉庫場所マスタ
| 項目 | 内容 |
| --- | --- |
| 物理名 | `stockkeeper.tm_warehouse_location` |
| 主キー | `warehouse_location_code` |
| 主要カラム | `warehouse_location_name`, `region_code`, `temperature_zone`, `active_flag` |
| 用途 | 倉庫場所の識別、地域・温度帯などの配送条件判定補助 |

### 7.2 在庫残高
| 項目 | 内容 |
| --- | --- |
| 物理名 | `stockkeeper.t_stock_balance` |
| 主キー | `item_code, warehouse_location_code` |
| 主要カラム | `on_hand_quantity`, `reserved_quantity`, `version_no`, `updated_at` |
| 用途 | 倉庫場所ごとの保有在庫数、引当済在庫数、排他制御用バージョン管理 |

### 7.3 在庫トランザクション履歴
| 項目 | 内容 |
| --- | --- |
| 物理名 | `stockkeeper.th_stock_transaction` |
| 主キー | `stock_tx_id` |
| 主要カラム | `reservation_id`, `order_id`, `order_line_no`, `item_code`, `warehouse_location_code`, `tx_type`, `quantity`, `before_on_hand_quantity`, `before_reserved_quantity`, `after_on_hand_quantity`, `after_reserved_quantity`, `occurred_at` |
| 用途 | 引当、引当解除、出荷確定の監査履歴 |

## 8. OrderHub 側への保持方針
- `orderhub.t_stock_reservation_result` に `warehouse_location_code` を保持する。
- `orderhub.t_shipment_request` に `source_warehouse_location_code` を保持する。
- OrderHub 側では在庫正本を持たず、引当・解除・出荷確定の結果スナップショットのみを保持する。
