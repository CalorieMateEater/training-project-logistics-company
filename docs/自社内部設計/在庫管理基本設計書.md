# 在庫管理基本設計書

## 1. 目的
本書は、Hoge Stock Keeper における在庫数量モデル、倉庫場所管理、倉庫担当者の権限制御、在庫ライフサイクル、排他制御、主要テーブルを定義し、OrderHub および倉庫担当者端末から参照される在庫管理の基本設計を整理する。

## 2. 設計方針
- 在庫は `商品コード + 倉庫場所コード` 単位で管理する。
- 数量は `保有在庫数` と `引当済在庫数` を分けて保持し、`利用可能在庫数 = 保有在庫数 - 引当済在庫数` とする。
- 引当成功時は保有在庫数を減算せず、引当済在庫数のみを加算する。
- 引当解除時は引当済在庫数のみを減算する。
- 出荷確定時は保有在庫数と引当済在庫数を同時に減算する。
- 入庫登録時は保有在庫数と利用可能在庫数のみを加算し、引当済在庫数は変更しない。
- 倉庫担当者の参照・登録権限は `employee_id + warehouse_location_code` 単位で管理する。
- 入庫登録の重複防止は `warehouse_location_code + receipt_reference_no` 単位で制御する。
- 将来の配送会社選定に備え、引当結果には倉庫場所コードを含めて OrderHub へ返却する。

## 3. 在庫ライフサイクル

### 3.1 数量定義
| 項目 | 説明 |
| --- | --- |
| 保有在庫数 | Hoge社が対象倉庫場所に物理保有している数量 |
| 引当済在庫数 | 受注により予約済で、まだ出荷確定していない数量 |
| 利用可能在庫数 | `保有在庫数 - 引当済在庫数` |

### 3.2 業務イベントと数量変化
| イベント | 保有在庫数 | 引当済在庫数 | 備考 |
| --- | --- | --- | --- |
| 入庫登録 | 増加 | 変化なし | 倉庫担当者が実入庫を反映する |
| 在庫引当 | 変化なし | 増加 | 倉庫場所を確定する |
| 在庫引当解除 | 変化なし | 減少 | 未出荷取消、障害復旧時 |
| 在庫出荷確定 | 減少 | 減少 | 初版では Bar社の `配送受付済` を契機とする |

## 4. 倉庫場所設計
| 項目 | 内容 |
| --- | --- |
| 管理単位 | `warehouse_location_code` |
| 想定用途 | 在庫引当先の保持、将来の配送会社選定、配送制約判定、拠点別在庫分析 |
| 初版での扱い | 1注文明細につき1倉庫場所を返却する |
| OrderHub への連携 | 在庫引当結果と出荷依頼に倉庫場所コードを保持する |

## 5. 倉庫担当者権限制御
| 項目 | 内容 |
| --- | --- |
| 管理単位 | `employee_id + warehouse_location_code` |
| 許可内容 | 在庫照会、入庫登録 |
| 権限制約 | 担当外倉庫場所への照会・登録は `403 Forbidden` |
| 複数担当 | 1倉庫場所に複数担当者を割り当て可能 |

## 6. API設計要点
| IF-ID | API | 主処理 |
| --- | --- | --- |
| `IF-HOGE-STK-001` | 在庫引当API | 利用可能在庫を確認し、倉庫場所を確定して引当済在庫数を更新 |
| `IF-HOGE-STK-002` | 在庫引当解除API | 指定引当を解除し、引当済在庫数を戻す |
| `IF-HOGE-STK-003` | 在庫出荷確定API | 指定引当を出荷確定し、保有在庫数と引当済在庫数へ反映 |
| `IF-HOGE-STK-004` | 倉庫在庫照会API | 担当倉庫場所に対する在庫残高一覧を返却 |
| `IF-HOGE-STK-005` | 入庫登録API | 実入庫を反映し、入庫受付番号で冪等制御する |

## 7. 排他制御方針
- 排他制御単位は `商品コード + 倉庫場所コード` とする。
- 在庫残数更新時は在庫残高テーブルに対して排他ロックを取得する。
- 実装方式は行ロック、または `version_no` による楽観ロックを許容するが、初版基本設計では `version_no` を保持する。
- 排他待ちタイムアウト時は再送可能異常として OrderHub へ返却する。

## 8. テーブル設計

### 8.1 倉庫場所マスタ
| 項目 | 内容 |
| --- | --- |
| 物理名 | `stockkeeper.tm_warehouse_location` |
| 主キー | `warehouse_location_code` |
| 主要カラム | `warehouse_location_name`, `region_code`, `temperature_zone`, `active_flag` |
| 用途 | 倉庫場所の識別、地域・温度帯などの配送条件判定補助 |

### 8.2 在庫商品マスタ
| 項目 | 内容 |
| --- | --- |
| 物理名 | `stockkeeper.tm_stock_item` |
| 主キー | `item_code` |
| 主要カラム | `item_name`, `temperature_zone`, `size_type`, `active_flag` |
| 用途 | 入庫対象商品の妥当性確認、将来の配送条件判定補助 |

### 8.3 倉庫担当者スコープ
| 項目 | 内容 |
| --- | --- |
| 物理名 | `stockkeeper.tm_warehouse_staff_scope` |
| 主キー | `employee_id, warehouse_location_code` |
| 主要カラム | `employee_name`, `role_code`, `active_flag` |
| 用途 | 倉庫担当者の担当倉庫場所管理 |

### 8.4 在庫残高
| 項目 | 内容 |
| --- | --- |
| 物理名 | `stockkeeper.t_stock_balance` |
| 主キー | `item_code, warehouse_location_code` |
| 主要カラム | `on_hand_quantity`, `reserved_quantity`, `last_received_at`, `version_no`, `updated_at` |
| 用途 | 倉庫場所ごとの保有在庫数、引当済在庫数、排他制御用バージョン管理 |

### 8.5 入庫受付履歴
| 項目 | 内容 |
| --- | --- |
| 物理名 | `stockkeeper.th_stock_receipt_history` |
| 主キー | `warehouse_location_code, receipt_reference_no` |
| 主要カラム | `employee_id`, `item_code`, `received_quantity`, `after_on_hand_quantity`, `after_available_quantity`, `registered_at` |
| 用途 | 入庫受付番号の重複防止、および入庫登録監査 |

### 8.6 在庫トランザクション履歴
| 項目 | 内容 |
| --- | --- |
| 物理名 | `stockkeeper.th_stock_transaction` |
| 主キー | `stock_tx_id` |
| 主要カラム | `reservation_id`, `receipt_reference_no`, `order_id`, `order_line_no`, `item_code`, `warehouse_location_code`, `tx_type`, `quantity`, `before_on_hand_quantity`, `before_reserved_quantity`, `after_on_hand_quantity`, `after_reserved_quantity`, `occurred_at` |
| 用途 | 入庫、引当、引当解除、出荷確定の監査履歴 |

## 9. OrderHub 側への保持方針
- `orderhub.t_stock_reservation_result` に `warehouse_location_code` を保持する。
- `orderhub.t_shipment_request` に `source_warehouse_location_code` を保持する。
- OrderHub 側では在庫正本を持たず、引当・解除・出荷確定の結果スナップショットのみを保持する。
