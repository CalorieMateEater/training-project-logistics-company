# 11. バリデーション仕様書

## 1. 文書目的

本書は、`Hoge OrderHub` の `FooOrderImportJob` における入力ファイルバリデーション仕様を定義する。  
障害調査時には、ログに出力されたエラーコードと不正値を照合するための一次資料として利用する。

## 2. 対象ファイル

- IF-ID: `IF-FOO-HOGE-001`
- ファイル名: `FOO_ORDER_yyyyMMddHHmmssSSS_NNN.dat`

## 3. 項目別バリデーション

| 項目 | 条件 | エラーコード | エラーメッセージ |
| --- | --- | --- | --- |
| record_type | `D` 固定 | E-HOGE-VAL-001 | レコード種別が不正です |
| partner_order_id | 必須、半角英数字14桁 | E-HOGE-VAL-002 | 提携先注文IDが不正です |
| order_type | `01`、`02`、`03` のいずれか | E-HOGE-VAL-003 | 注文区分が不正です |
| priority_level | `0`〜`9` の整数 | E-HOGE-VAL-010 | 優先度が不正です |
| priority_level | `order_type=02` 以外は `0` 固定 | E-HOGE-VAL-011 | 予約注文以外で優先度を指定できません |
| customer_id | 必須、半角英数字12桁 | E-HOGE-VAL-004 | 顧客IDが不正です |
| item_code | 必須、半角英数字10桁 | E-HOGE-VAL-005 | 商品コードが不正です |
| quantity | 1以上999以下の整数 | E-HOGE-VAL-006 | 数量が不正です |
| order_datetime | `yyyy-MM-ddTHH:mm:ss` 形式 | E-HOGE-VAL-007 | 注文日時が不正です |
| delivery_zip_code | 半角数字7桁 | E-HOGE-VAL-008 | 配送先郵便番号が不正です |
| delivery_address | 必須、200文字以内 | E-HOGE-VAL-009 | 配送先住所が不正です |
| shipping_release_at | 指定時は `order_type=02` のみ可 | E-HOGE-VAL-012 | 出荷解放日時を指定できるのは予約注文のみです |
| shipping_release_at | 指定時は `order_datetime` より後の `yyyy-MM-ddTHH:mm:ss` 形式 | E-HOGE-VAL-013 | 出荷解放日時が不正です |

## 4. エラー判定ルール

- 1レコード内に複数項目の不正がある場合、それぞれ個別にエラーを記録する
- バリデーションエラーが1件でも存在するファイルは異常扱いとする
- 異常ファイルはエラーディレクトリへ退避する
- バリデーションエラー発生時は、後続のAPI呼び出し、DB登録、メッセージ送信は実施しない

## 5. 調査時の見方

異常時ログに以下が出力されている場合、本仕様書と照合して入力データ不正であると判断できる。

- `errorCode=E-HOGE-VAL-003`
- `field=order_type`

現行実装ログでは、エラーコードではなく `order_type invalid` のような理由文字列だけが出力される場合がある。
その場合も、対象項目と理由文字列を本仕様書の条件へ照合する。
具体的な不正値は、ログおよび対象ファイルの該当行を確認する。
