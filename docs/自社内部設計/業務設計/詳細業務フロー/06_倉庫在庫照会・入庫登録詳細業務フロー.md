# DFL-006 倉庫在庫照会・入庫登録詳細業務フロー

## 1. 目的
Hoge社倉庫担当者による在庫照会と入庫登録について、認証、担当倉庫場所の権限制御、Stock Keeper 内部テーブルの CRUD、重複防止までを詳細化する。

## 2. 設計書ID
| 項目 | 内容 |
| --- | --- |
| 設計書ID | `DFL-006` |
| 業務領域 | 倉庫在庫運用 |
| 逆引き対象処理設計書 | `PDS-011`, `PDS-012` |

## 3. 登場アクター・内部コンポーネント
- Hoge社倉庫担当者
- 倉庫担当者端末
- 在庫照会・入庫登録API
- Stock Keeper データストア

## 4. 詳細業務フロー図
```mermaid
sequenceDiagram
    box rgb(237, 246, 255) 自社
        actor WarehouseUser as Hoge社倉庫担当者
        participant Terminal as 倉庫担当者端末
        participant StockApi as 在庫照会・入庫登録API
        participant StockDb as Stock Keeperデータストア
    end

    alt 在庫照会
        WarehouseUser->>Terminal: 倉庫場所・商品条件を指定
        Terminal->>StockApi: 在庫照会API呼出
        StockApi->>StockDb: 倉庫担当者スコープ R
        StockApi->>StockDb: 倉庫場所 / 在庫商品 / 在庫残高 R
        StockApi-->>Terminal: 在庫一覧応答
        Terminal-->>WarehouseUser: 在庫表示
    else 入庫登録
        WarehouseUser->>Terminal: 入庫情報入力(倉庫場所、商品、数量、受付番号)
        Terminal->>StockApi: 入庫登録API呼出
        StockApi->>StockDb: 倉庫担当者スコープ R
        StockApi->>StockDb: 倉庫場所 / 在庫商品 / 入庫受付履歴 R
        alt 同一受付番号・同一内容
            StockApi->>StockDb: 入庫受付履歴 R
            StockApi-->>Terminal: 既存登録結果返却
        else 初回登録
            StockApi->>StockDb: 在庫残高 R/U または C
            StockApi->>StockDb: 入庫受付履歴 C
            StockApi->>StockDb: 在庫トランザクション履歴 C
            StockApi-->>Terminal: 更新後在庫応答
        else 同一受付番号・異内容
            StockApi-->>Terminal: 409 応答
        end
        Terminal-->>WarehouseUser: 登録結果表示
    end
```

## 5. 処理単位と CRUD
| 処理単位 | 主体 | 主な DB CRUD | 補足 |
| --- | --- | --- | --- |
| 倉庫スコープ確認 | 在庫照会・入庫登録API | 倉庫担当者スコープ `R` | `employee_id` と `warehouse_location_code` の対応を確認する |
| 在庫照会 | 在庫照会・入庫登録API | 倉庫場所 `R`、在庫商品 `R`、在庫残高 `R` | `item_code` 未指定時は担当倉庫の登録済在庫を一覧返却する |
| 入庫重複判定 | 在庫照会・入庫登録API | 入庫受付履歴 `R` | `warehouse_location_code + receipt_reference_no` で照合する |
| 入庫確定 | 在庫照会・入庫登録API | 在庫商品 `R`、在庫残高 `R/U/C`、入庫受付履歴 `C`、在庫トランザクション履歴 `C` | 保有在庫数と利用可能在庫数のみ増加させる |

## 6. 詳細ルール
- 倉庫担当者は、自身に割り当てられた担当倉庫場所のみ照会・登録できる。
- 在庫照会APIは、対象在庫が存在しない場合は 200 応答で空配列を返す。
- 入庫登録APIは、在庫残高が未作成でも有効な商品コード・倉庫場所コードであれば残高を新規作成する。
- 同一 `receipt_reference_no` の同一内容再送は重複成功として扱い、最初の登録結果を返す。
- 同一 `receipt_reference_no` で商品コードまたは数量が異なる場合は `409 Conflict` とする。
