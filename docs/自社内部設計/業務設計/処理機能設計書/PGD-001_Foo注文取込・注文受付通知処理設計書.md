# PGD-001 Foo注文取込・注文受付通知処理設計書

## 1. 基本情報
| 項目 | 内容 |
| --- | --- |
| 処理機能ID | `PGD-001` |
| 関連処理設計書ID | `PDS-001`, `PDS-002` |
| 論理機能名 | Foo注文取込・注文受付通知 |
| 概要 | Foo社から到着した注文ファイルを取り込み、注文登録、在庫・顧客確認、配送依頼待ち登録、注文受付通知対象の登録までを行う。 |

## 2. 入出力
| 区分 | 内容 |
| --- | --- |
| 入力 | Foo注文ファイル、顧客確認結果、在庫引当結果 |
| 出力 | 注文ヘッダ、注文明細、出荷依頼、注文受付通知履歴 |
| 後続 | 注文受付通知処理、配送会社連携処理 |

## 3. 主処理
1. 注文ファイルを1レコードずつ検証する。
2. 顧客確認と在庫引当を実施する。
3. 注文元、優先度、配送解放時刻を含めて注文情報を登録する。
4. 配送会社連携用の送信待ち情報を登録する。
5. 注文受付通知用の通知履歴を `PENDING` で登録する。

## 4. 制御方針
- 予約注文で配送解放時刻が未来の場合は `WAITING_SHIPPING_RELEASE` とする。
- Bar社営業時間外の即時送信は行わず、後続の配送会社連携処理へ引き継ぐ。
- 注文受付通知ファイルの物理出力は本機能では行わず、通知処理へ委譲する。
