# PGD-008 未出荷取消処理設計書

## 1. 基本情報
| 項目 | 内容 |
| --- | --- |
| 処理機能ID | `PGD-008` |
| 関連処理設計書ID | `PDS-010` |
| 論理機能名 | 未出荷取消 |
| 概要 | 未出荷状態の注文を取消し、引当済在庫を解放したうえで、配送会社向け送信待ちを停止する。 |

## 2. 入出力
| 区分 | 内容 |
| --- | --- |
| 入力 | 注文ID、取消理由、呼出元識別子 |
| 出力 | 取消結果、更新後注文状態、IF履歴 |

## 3. 主処理
1. 呼出元が内部運用クライアントであることを確認する。
2. 注文状態と出荷依頼状態が取消可能条件を満たすことを確認する。
3. 引当済在庫を在庫管理APIで解放する。
4. 注文状態と出荷依頼状態を `CANCELLED` へ更新する。
5. IF履歴へ取消結果を記録する。

## 4. 制御方針
- すでに `RELEASED` の在庫引当結果は再解放しない。
- 注文状態または出荷依頼状態が取消不可であれば `409 Conflict` とする。
- 注文または出荷依頼が見つからない場合は `404 Not Found` とする。
