# PGD-004 Hoge直受注登録処理設計書

## 1. 基本情報
| 項目 | 内容 |
| --- | --- |
| 処理機能ID | `PGD-004` |
| 関連処理設計書ID | `PDS-007` |
| 論理機能名 | Hoge直受注登録 |
| 概要 | Hoge社業務部門の直受注登録APIを受け付け、入力検証、顧客確認、在庫引当、配送会社送信待ち登録までを行う。 |

## 2. 入出力
| 区分 | 内容 |
| --- | --- |
| 入力 | 直受注登録APIリクエスト |
| 出力 | 注文情報、配送会社送信待ち情報、受付応答 |

## 3. 主処理
1. 呼出元識別子とAPI入力値を検証する。
2. 重複依頼を検出する。
3. 顧客確認と在庫引当を実施する。
4. `order_source=HOGE`、`partner_priority_level=0`、`shipping_priority_class=NORMAL` として注文情報と配送会社送信待ち情報を登録する。
5. 在庫引当成功後に注文確定前の後続処理が失敗した場合は、当該引当を補償解除する。
6. 現在状態を含む受付応答を返却する。

## 4. 制御方針
- Hoge社直受注の `partner_request_id` を一意制御する。
- `shipping_release_at` 未到来の場合は配送会社を問わず出荷解放待ちとする。
- 配送会社がBarの場合は営業時間外待機、Fugaの場合は即時送信対象とする。
