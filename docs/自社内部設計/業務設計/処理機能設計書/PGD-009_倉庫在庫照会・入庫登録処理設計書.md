# PGD-009 倉庫在庫照会・入庫登録処理設計書

## 1. 基本情報
| 項目 | 内容 |
| --- | --- |
| 処理機能ID | `PGD-009` |
| 関連処理設計書ID | `PDS-011`, `PDS-012` |
| 論理機能名 | 倉庫在庫照会・入庫登録 |
| 概要 | 倉庫担当者が自身の担当倉庫場所の在庫を参照し、実入庫発生時に Stock Keeper へ即時反映する。 |

## 2. 入出力
| 区分 | 内容 |
| --- | --- |
| 入力 | 在庫照会APIリクエスト、入庫登録APIリクエスト |
| 出力 | 在庫照会APIレスポンス、入庫登録APIレスポンス |

## 3. 主処理
1. 倉庫担当者の認証情報から `employee_id` を特定する。
2. 対象倉庫場所が担当スコープに含まれるか確認する。
3. 在庫照会では在庫残高を参照し、一覧応答を編集する。
4. 入庫登録では `receipt_reference_no` の重複判定を行う。
5. 重複なしの場合のみ、保有在庫数と利用可能在庫数を更新し、監査履歴を記録する。

## 4. 制御方針
- 倉庫担当者は担当倉庫場所のみ照会・登録可能とする。
- 入庫登録は `warehouse_location_code + receipt_reference_no` 単位で冪等制御する。
- 入庫登録時に在庫残高が存在しない場合は新規作成する。
- 入庫で増加させるのは保有在庫数、利用可能在庫数のみとし、引当済在庫数は変更しない。
