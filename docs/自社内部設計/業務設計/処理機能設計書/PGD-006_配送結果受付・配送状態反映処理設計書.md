# PGD-006 配送結果受付・配送状態反映処理設計書

## 1. 基本情報
| 項目 | 内容 |
| --- | --- |
| 処理機能ID | `PGD-006` |
| 関連処理設計書ID | `PDS-004`, `PDS-005` |
| 論理機能名 | 配送結果受付・配送状態反映 |
| 概要 | Bar社またはFuga社から配送結果を受信し、重複判定、履歴登録、最新状態更新、Bar社配送受付済時の在庫出荷確定、後続通知登録までを行う。 |

## 2. 入出力
| 区分 | 内容 |
| --- | --- |
| 入力 | 配送結果受付APIリクエスト |
| 出力 | 配送状態履歴、最新配送状態、後続通知履歴 |

## 3. 主処理
1. 対象注文の存在確認を行う。
2. 状態連番で重複・旧着を判定する。
3. Bar社の初回 `配送受付済` 反映時のみ在庫出荷確定を実行する。
4. 配送状態履歴を登録する。
5. 最新配送状態と注文状態を更新する。
6. 注文元や状態種別に応じて後続通知を登録する。

## 4. 制御方針
- 旧着・重複状態は受付済としてスキップし、再反映しない。
- Bar社の `配送受付済` の初回反映時のみ、在庫引当結果を出荷確定へ更新する。
- Foo注文では配送結果返却通知を登録する。
- 配送完了時は請求確定通知を追加登録する。
