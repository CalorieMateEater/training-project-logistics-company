# データモデル設計書

## 1. 目的
Hoge OrderHub が保持する業務データの主なエンティティ、主キー、関係、更新責務を整理する。

## 2. 設計方針
- 業務上の正本データは OrderHub データストアで管理する。
- 外部連携の送受信記録は、業務データとは分離した連携履歴エンティティで保持する。
- 配送結果は最新状態と履歴を分けて保持し、障害調査と監査追跡を両立する。
- 冪等制御に必要なキーは業務トランザクションと同一データストアで管理する。

## 3. エンティティ一覧
| エンティティ | 概要 | 主キー |
| --- | --- | --- |
| 注文ヘッダ | 受注の代表情報 | `order_id` |
| 注文明細 | 受注商品の明細 | `order_id + order_line_no` |
| 出荷依頼 | 配送会社への出荷依頼単位 | `shipment_request_id` |
| 配送状態最新 | 注文ごとの現在配送状態 | `order_id` |
| 配送状態履歴 | Bar社またはFuga社から受信した状態変化履歴 | `order_id + status_seq` |
| 顧客確認結果 | 顧客確認 API 呼出結果 | `order_id` |
| 在庫引当結果 | 在庫引当 API 呼出結果 | `order_id + order_line_no` |
| 通知履歴 | Baz社、Qux社、Foo社返却の通知記録 | `notification_id` |
| 連携履歴 | 外部 IF 単位の送受信記録 | `interface_history_id` |
| 冪等受付履歴 | 配送会社向け送信時の再送制御記録 | `idempotency_key` |
| アーカイブ管理 | アーカイブ出力単位の管理情報 | `archive_execution_id` |

## 4. 主要エンティティ

### 4.1 注文ヘッダ
| 項目 | 概要 |
| --- | --- |
| `order_id` | OrderHub 内部注文 ID |
| `partner_order_id` | 提携先側注文 ID |
| `partner_request_id` | Hoge直受注登録時の内部受付要求ID |
| `order_source` | 受注元種別。`FOO`, `HOGE` |
| `partner_priority_level` | 提携先から受けた優先度。Foo注文のみ保持 |
| `shipping_priority_class` | 配送会社向け送信時の優先配送区分。`NORMAL`, `PRIORITY` |
| `customer_id` | 顧客 ID |
| `delivery_zip_code` | 注文受付時の配送先郵便番号スナップショット |
| `delivery_address` | 注文受付時の配送先住所スナップショット |
| `delivery_name` | 注文受付時の配送先氏名スナップショット |
| `delivery_phone` | 注文受付時の配送先電話番号スナップショット |
| `package_count` | 配送会社へ依頼する荷物個数 |
| `payment_method` | 支払方法。`PREPAID`, `COD` |
| `requested_delivery_date` | 配送希望日 |
| `special_instruction` | 配送上の特記事項 |
| `subtotal_excluding_tax` | 明細税抜金額の合計 |
| `tax_amount` | 明細単位で算出した消費税額の合計 |
| `billing_amount` | 税込請求金額 |
| `order_status` | 受注状態 |
| `shipment_status` | 配送状態 |
| `carrier_code` | 選定された配送会社コード |
| `shipping_release_at` | 予約出荷の解放日時 |
| `created_at` | 注文登録日時 |
| `updated_at` | 最終更新日時 |

## 5. エンティティ関連
```text
注文ヘッダ 1 --- n 注文明細
注文ヘッダ 1 --- 0..1 出荷依頼
注文ヘッダ 1 --- 1 配送状態最新
注文ヘッダ 1 --- n 配送状態履歴
注文ヘッダ 1 --- n 通知履歴
注文ヘッダ 1 --- n 連携履歴
出荷依頼 1 --- 0..n 冪等受付履歴
```

## 6. 更新責務
| エンティティ | 更新主体 |
| --- | --- |
| 注文ヘッダ | FooOrderImportJob、出荷依頼受付API、配送状態取込Worker |
| 注文明細 | FooOrderImportJob、出荷依頼受付API |
| 出荷依頼 | FooOrderImportJob、出荷依頼受付API、配送会社連携Worker |
| 配送状態最新 | 配送状態取込Worker |
| 配送状態履歴 | 配送状態取込Worker |
| 顧客確認結果 | FooOrderImportJob、出荷依頼受付API |
| 在庫引当結果 | FooOrderImportJob、出荷依頼受付API、配送状態取込Worker、運用取消処理 |
| 通知履歴 | 配送会社連携Worker、配送状態取込Worker、配送結果返却Worker |
| 連携履歴 | 全 IF 呼出処理 |
| 冪等受付履歴 | 配送会社連携Worker |
| アーカイブ管理 | 日次アーカイブBatch |

## 7. 索引方針
- 注文ヘッダ: `partner_order_id`, `order_status`, `shipment_status`, `created_at`
- 出荷依頼: `shipment_request_id`, `carrier_accept_no`, `shipment_request_status`, `shipping_priority_class`, `source_warehouse_location_code`
- 配送状態履歴: `order_id`, `status_seq`, `event_occurred_at`
- 連携履歴: `if_id`, `request_datetime`, `result_code`

## 8. 保持期間
| データ | 保持期間 |
| --- | --- |
| 注文ヘッダ、注文明細 | 3年 |
| 配送状態履歴 | 3年 |
| 通知履歴 | 1年 |
| 連携履歴 | 1年 |
| 冪等受付履歴 | 90日 |
