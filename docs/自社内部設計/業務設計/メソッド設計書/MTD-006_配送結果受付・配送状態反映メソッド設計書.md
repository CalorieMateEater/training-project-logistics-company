## 3. `accept(...)`
### 3.1 シグネチャ
```java
public void accept(BarDeliveryResultRequest request)
```

### 3.2 処理概要
1. 注文番号と配送会社受付番号の整合性を確認する。
2. 配送結果コードをHoge社内部状態へマッピングする。
3. 初回の `配送受付済` 反映時は在庫出荷確定を実行する。
4. 現在配送状態を更新し、履歴を追加する。
5. Foo返却対象の通知履歴を起票する。
6. IF履歴を記録する。

### 3.3 フロー図
```mermaid
flowchart TD
    A[配送結果受信] --> B[対象注文検索]
    B --> C[結果コード変換]
    C --> D{初回配送受付済か}
    D -->|Yes| E[在庫出荷確定]
    D -->|No| F[現在配送状態更新]
    E --> F
    F --> G[配送状態履歴追加]
    G --> H[Foo返却通知起票]
    H --> I[IF履歴記録]
```
