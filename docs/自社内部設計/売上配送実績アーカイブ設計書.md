# 10. S3オブジェクト設計書 Hoge Sales Archive

## 1. 対象IF

- IF-ID: `IF-HOGE-ARC-001`
- IF名: 売上・配送実績オブジェクト保管

## 2. 基本情報

| 項目 | 内容 |
| --- | --- |
| IF-ID | IF-HOGE-ARC-001 |
| オブジェクト名 | `HOGE_SALES_yyyyMMdd.csv` |
| 送信元 | Hoge OrderHub 日次アーカイブBatch |
| 送信先 | Hoge Sales Archive |
| 方式 | Amazon S3 |
| 保管経路 | OrderHub データストア → 日次アーカイブBatch → アーカイブ出力S3 → 売上アーカイブS3 |
| 文字コード | UTF-8 |
| 改行コード | LF |
| 区切り文字 | カンマ |
| 作成タイミング | 完了済み出荷依頼を対象とした日次バッチ |
| 一時出力先 | `s3://hoge-archive-export/sales/` |
| 保管先 | `s3://hoge-sales-archive/sales/` |

## 3. レコード形式

```text
sales_date,order_id,customer_id,item_code,quantity,sales_amount,delivery_status,delivery_company_code,billing_linked_at,notice_linked_at
```

## 4. 項目定義

| No | 項目名 | 物理名 | 型 | 必須 | 桁数 | 説明 |
| ---: | --- | --- | --- | --- | ---: | --- |
| 1 | 売上日 | sales_date | string | 必須 | 10 | yyyy-MM-dd |
| 2 | 注文ID | order_id | string | 必須 | 21 | Hoge OrderHub注文ID |
| 3 | 顧客ID | customer_id | string | 必須 | 12 | 顧客ID |
| 4 | 商品コード | item_code | string | 必須 | 10 | 商品コード |
| 5 | 数量 | quantity | number | 必須 | 1〜999 | 数量 |
| 6 | 売上金額 | sales_amount | number | 必須 | 1〜9999999 | 売上金額 |
| 7 | 配送状態 | delivery_status | string | 必須 | 20 | アーカイブ時点の配送状態 |
| 8 | 配送会社コード | delivery_company_code | string | 必須 | 10 | `BAR` または `FUGA` |
| 9 | 請求連携日時 | billing_linked_at | string | 任意 | 19 | yyyy-MM-ddTHH:mm:ss |
| 10 | 通知連携日時 | notice_linked_at | string | 任意 | 19 | yyyy-MM-ddTHH:mm:ss |

## 5. 補足

本IFは `Hoge社内のS3保管連携` であり、外部会社とのHULFT連携ではない。  
入力ファイル受信とは方式も方向も異なるため、障害調査時に混同しないこと。

補足として、日次アーカイブBatch は OrderHub データストアから抽出した完了済み出荷依頼、配送結果、請求連携履歴、通知履歴をCSVオブジェクトとして `アーカイブ出力S3` へ出力し、その後 `売上アーカイブS3` に保管する。
