## 4. バー社連携の要点

- `出荷指示登録API` は Hoge社からバー社への同期APIである
- `配送結果返却API` は バー社からHoge社への同期コールバックAPIである
- バー社の `出荷指示登録API` では、サーバ側で冪等性制御を行う
- 冪等性判定は `X-Idempotency-Key` と業務キーの両方を用いて行い、単純な重複POSTと内容差し替えPOSTを区別する
- Hoge社からバー社へ送信する電文には、注文元コードと優先配送区分が含まれる
- Foo社予約注文で優先配送対象となる場合のみ、Bar向け電文の優先配送区分を `PRIORITY` とする
- 配送結果返却では、同一配送案件に対する複数イベント通知と順序管理を考慮する
