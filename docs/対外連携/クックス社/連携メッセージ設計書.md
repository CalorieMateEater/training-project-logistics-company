# 08. 電文設計書 Qux Mall Sync

## 1. 対象IF

- IF-ID: `IF-HOGE-QUX-001`
- IF名: 注文状態通知メッセージ

## 2. 基本情報

| 項目 | 内容 |
| --- | --- |
| キュー名 | order-notice-queue.fifo |
| 送信元 | Hoge OrderHub |
| 送信先 | Qux Mall Sync |
| 方式 | SQS FIFO Queue |
| MessageGroupId | order_id |
| 送信タイミング | Bar向け出荷依頼送信成功後 |
| メッセージ形式 | JSON |
| 実行主体 | Hoge OrderHub Worker |
| キュー配置 | Hoge社AWSアカウント |

## 3. メッセージ例

```json
{
  "event_type": "STATUS_UPDATED",
  "notification_key": "qux-status:O20260620142015A1B2C3:3",
  "order_id": "O20260620142015A1B2C3",
  "external_order_id": "FO202606080001",
  "requested_at": "2026-06-08T02:00:05",
  "status": "PREPARING_FOR_SHIPMENT",
  "display_status_name": "住所補正対応中"
}
```

## 4. 項目定義

| 項目名 | 物理名 | 型 | 必須 | 桁数 | 説明 |
| --- | --- | --- | --- | ---: | --- |
| イベント種別 | event_type | string | 必須 | 30 | `SHIPMENT_REQUESTED`, `STATUS_UPDATED` |
| 通知キー | notification_key | string | 必須 | 128 | 重複抑止用キー |
| 注文ID | order_id | string | 必須 | 21 | Hoge OrderHubの注文ID |
| 外部注文ID | external_order_id | string | 必須 | 14 | Foo社連携元注文ID |
| 送信日時 | requested_at | string | 必須 | 19 | yyyy-MM-ddTHH:mm:ss |
| 状態 | status | string | 必須 | 20 | Hoge社管理の現在状態 |
| 表示用状態名称 | display_status_name | string | 任意 | Qux社画面向けの表示名称 |

## 5. キュー運用補足

- Workerは `order-notice-queue.fifo` にメッセージを投入する
- Qux Mall Sync は Consumer として順序保証付きで取り込む
- Hoge社は夜間通知を抑止せず即時送信し、Qux社側で表示タイミングを制御する
- 失敗時の再処理や例外運用は本資料では詳細化しない

## 6. 補足

- `Qux Mall Sync` は外部モール向けの通知サービスである
- 注文登録未完了、またはBar向け出荷依頼送信前に停止した場合、本メッセージは未送信となる
