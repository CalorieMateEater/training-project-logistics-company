# リリース設計書

## 1. 目的
Hoge社受注連携基盤におけるアプリケーションおよび基盤変更のリリース手順、承認、切戻し方針を整理する。

## 2. 対象
- ECS アプリケーション更新
- Infrastructure as Code 更新
- API 設定変更
- HULFT 設定変更

## 3. リリース方式
| 対象 | 方式 |
| --- | --- |
| 外部 API | Blue/Green |
| 内部 API | Rolling |
| Worker | Rolling |
| Batch | 次回実行反映 |
| HULFT | 設定反映後疎通確認 |

## 4. 標準手順
1. 変更申請起票
2. レビュー、承認
3. 検証環境反映
4. IF 疎通確認
5. 本番反映
6. 監視確認
7. 完了報告

## 5. 切戻し方針
| 対象 | 切戻し方法 |
| --- | --- |
| ECS | 旧タスク定義へ戻す |
| Terraform | 直前タグへ戻して再適用 |
| Secrets / Parameter | 前バージョンを再指定 |
| HULFT | 旧設定バックアップから復元 |

## 6. 切戻し判断
- API 5xx 継続
- 重要 IF 疎通不可
- Foo 注文取込不可
- Bar/Fuga 結果反映不可
- SQS 通知投入不可
