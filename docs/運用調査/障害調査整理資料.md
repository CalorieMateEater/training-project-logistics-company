# 13. 障害調査整理資料

## 1. 文書目的

本書は、障害調査結果、報告文例、および確認項目を整理する資料である。

## 2. 調査結果

### 2.1 障害概要

`Hoge OrderHub` の `FooOrderImportJob` にて、`Foo Order Gateway` から受信した出荷依頼ファイルの取込処理中にバリデーションエラーが発生し、ジョブが異常終了した。

### 2.2 直接原因

`Foo Order Gateway` から HULFT 連携された出荷依頼ファイルに、`Hoge OrderHub` 側のファイル設計書で許可されていない値が含まれていた。

| 項目名 | 物理名 | 期待値 | 実際の値 | エラー内容 |
| --- | --- | --- | --- | --- |
| 注文区分 | order_type | `01`、`02`、`03` のいずれか | `99` | 定義外コード |
| 商品コード | item_code | 半角英数字10桁 | `ITEM-000012345` | 桁数超過、ハイフン混入 |

### 2.3 結論

- 原因は `Hoge OrderHub` のプログラム不具合ではない
- 原因は `Foo Order Gateway` から受信した入力データ不正である
- 調査依頼先は `Foo販売ソリューション株式会社` である

## 3. 報告文例

```text
1. エラー発生処理
Hoge OrderHub の FooOrderImportJob でエラーが発生しています。

2. 直接原因
Foo Order Gatewayから受信した出荷依頼ファイル `FOO_ORDER_20260608011540001_001.dat` の3行目でバリデーションエラーが発生しています。
エラー項目は order_type と item_code です。

3. 関係する外部システム
ファイル名、ファイル着信先S3、外部IF一覧より、対象ファイルは Foo Order Gateway から HULFT 連携されたものです。

4. 自社対応か他社依頼か
Hoge OrderHubのファイル設計書上、order_type は 01、02、03 のみ許可されています。
しかし実データは 99 でした。
また item_code は半角英数字10桁ですが、実データは ITEM-000012345 で、ハイフンを含み桁数も超過しています。
したがって、Hoge OrderHub側のプログラム不具合ではなく、Foo社からの連携データ不正と判断します。

5. 調査依頼先
Foo Order Gatewayの担当会社である Foo販売ソリューション株式会社 に調査依頼します。

依頼時に添付する情報は以下です。

- 発生日時: 2026-06-08 02:00
- 対象処理: Hoge OrderHub / FooOrderImportJob
- 対象ファイル: FOO_ORDER_20260608011540001_001.dat
- 対象行: 3行目
- 対象注文ID: FO202606080999
- エラー項目: order_type, item_code
- エラー値: order_type=99, item_code=ITEM-000012345
- エラーコード: E-HOGE-VAL-003, E-HOGE-VAL-005
```

## 4. 確認観点

| 確認項目 | 内容 |
| --- | --- |
| ログ確認 | FooOrderImportJobでエラーが発生していることを特定できる |
| エラー種別の理解 | API障害ではなくバリデーションエラーであることを理解できる |
| 入力元の特定 | FOO_ORDERファイルがFoo社由来であることを設計書から特定できる |
| 仕様違反の確認 | order_typeとitem_codeがファイル設計書に違反していることを説明できる |
| 他社依頼判断 | Foo販売ソリューション株式会社へ調査依頼すべきか判断できる |
| 依頼内容の具体性 | ファイル名、行番号、注文ID、項目名、エラー値を添えて依頼できる |
