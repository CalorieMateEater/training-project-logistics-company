# 07. 電文設計書 Baz Billing Hub

## 1. 対象IF

- IF-ID: `IF-HOGE-BAZ-001`
- IF名: 請求予定連携メッセージ

## 2. 基本情報

| 項目 | 内容 |
| --- | --- |
| キュー名 | billing-plan-queue |
| 送信元 | Hoge OrderHub |
| 送信先 | Baz Billing Hub |
| 方式 | SQS Standard Queue |
| 送信タイミング | Bar向け出荷依頼送信成功後 |
| メッセージ形式 | JSON |
| 実行主体 | Hoge OrderHub Worker |
| キュー配置 | Hoge社AWSアカウント |

## 3. メッセージ例

```json
{
  "event_type": "PROVISIONAL_BILLING",
  "notification_key": "baz-billing:O20260620142015A1B2C3:provisional",
  "reference_notification_id": null,
  "order_id": "O20260620142015A1B2C3",
  "customer_id": "C00000000001",
  "billing_amount": 12800,
  "tax_amount": 1163,
  "billing_date": "2026-06-08"
}
```

## 4. 項目定義

| 項目名 | 物理名 | 型 | 必須 | 桁数 | 説明 |
| --- | --- | --- | --- | ---: | --- |
| イベント種別 | event_type | string | 必須 | 30 | `PROVISIONAL_BILLING`, `FINAL_BILLING` |
| 通知キー | notification_key | string | 必須 | 128 | 再送・重複抑止用キー |
| 参照通知ID | reference_notification_id | string | 任意 | 訂正・確定時に元通知を参照する |
| 注文ID | order_id | string | 必須 | 21 | 注文ID |
| 顧客ID | customer_id | string | 必須 | 12 | 顧客ID |
| 請求金額 | billing_amount | number | 必須 | 1〜9999999 | 税込請求金額 |
| 消費税額 | tax_amount | number | 必須 | 0〜999999 | 消費税額 |
| 請求日 | billing_date | string | 必須 | 10 | yyyy-MM-dd |

## 5. バリデーション

| 項目 | 条件 |
| --- | --- |
| event_type | `PROVISIONAL_BILLING`, `FINAL_BILLING` のいずれか |
| billing_amount | 1以上 |
| billing_date | yyyy-MM-dd形式 |

## 6. 金額・日付編集規則

| 項目 | 編集規則 |
| --- | --- |
| `billing_amount` | 注文受付時に確定した明細税抜金額合計と消費税額合計の和 |
| `tax_amount` | 明細ごとに`floor(unit_price_excluding_tax × quantity × tax_rate ÷ 100)`で算出した消費税額の合計 |
| `billing_date`（仮請求） | Bar社の出荷依頼受付日。日本標準時の日付 |
| `billing_date`（確定請求） | 配送完了イベント発生日。日本標準時の日付 |

- 消費税の1円未満は明細単位で切り捨てる。
- 仮請求と確定請求は、注文受付時に保持した同一の金額スナップショットを使用する。
- 詳細は[業務データ・電文編集要件定義書](../../業務データ・電文編集要件定義書.md)を参照する。

## 7. キュー運用補足

- Workerはメッセージを `billing-plan-queue` に投入する
- Baz Billing Hub は Consumer としてキューをポーリングする
- Bar向け出荷依頼送信成功時は `PROVISIONAL_BILLING` を送信する
- 配送完了時は `FINAL_BILLING` を送信し、必要に応じて `reference_notification_id` で仮請求通知を参照する
- 失敗時の再処理や例外運用は本資料では詳細化しない

## 8. 補足

- `Baz Billing Hub` は外部決済サービスである
- 注文登録未完了、またはBar向け出荷依頼送信前に停止した場合、本メッセージは送信されない
