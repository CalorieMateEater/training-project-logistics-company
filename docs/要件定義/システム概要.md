# システム概要

## 1. 文書目的
本書は、Hoge社が提供する出荷・配送管理業務に関係するシステム群の全体像を整理し、対象業務、関係会社、システム間の役割分担を定義することを目的とする。

## 2. 対象範囲
- Hoge社の出荷・配送管理業務に関与する自社システム
- Foo社、Bar社、Fuga社、Baz社、Qux社の関連システム
- 直接連携を持たない補助外部システム

## 3. システムID命名方針

| 区分 | 命名方針 | 例 |
| --- | --- | --- |
| 自社システム | `Hoge SYS-*` | `Hoge SYS-01` |
| Foo社システム | `Foo SYS-*` | `Foo SYS-01` |
| Bar社システム | `Bar SYS-*` | `Bar SYS-01` |
| Fuga社システム | `Fuga SYS-*` | `Fuga SYS-01` |
| Baz社システム | `Baz SYS-*` | `Baz SYS-01` |
| Qux社システム | `Qux SYS-*` | `Qux SYS-01` |
| 補助外部システム | `quux SYS-*`、`corge SYS-*` | `quux SYS-01`、`corge SYS-01` |

## 5. 業務連携の全体像
本資料で扱う業務連携の全体像は以下のとおりである。

1. `Foo Order Gateway` が注文確定都度、HULFTで出荷依頼ファイルを `Hoge OrderHub` へ送信する
2. `Hoge OrderHub` が受信完了をトリガに出荷依頼を取り込み、顧客確認と在庫引当を行う
3. `Hoge OrderHub` が配送条件判定を行い、Bar社営業時間外は送信待ちキューへ保持した上で `Bar Delivery Center` へ配送指示を送信する
4. `Bar Delivery Center` が配送依頼受付、配送準備、配送中、配送完了などの配送イベントを時間差で `Hoge OrderHub` へ通知する
5. `Hoge OrderHub` が配送状態を反映し、`Foo Order Gateway` へHULFTで配送状態を返却する
6. `Foo Status Client` と `Fuga API Client` が必要に応じて `Hoge Shipping Gateway` のAPIで出荷状態を照会する
7. `Fuga API Client` は `Hoge Shipping Gateway` の `出荷依頼受付API` を呼び出して出荷依頼を登録する
8. `Hoge OrderHub` が `Baz` と `Qux` へ非同期連携する
9. `Hoge OrderHub` の `日次アーカイブBatch` が完了済みデータを `Hoge Sales Archive` の `S3` へ保管する
