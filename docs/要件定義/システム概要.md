# システム概要

## 1. 文書目的
本書は、Hoge社が提供する出荷・配送管理業務に関係するシステム群の全体像を整理し、対象業務、関係会社、システム間の役割分担を定義することを目的とする。

## 2. 対象範囲
- Hoge社の出荷・配送管理業務に関与する自社システム
- Foo社、Bar社、Fuga社、Baz社、Qux社の関連システム
- 直接連携を持たない補助外部システム

## 2.1 業務前提
- Hoge社は商品在庫を保有する荷主であり、自社在庫の在庫管理責任を負う。
- Hoge社は配送事業者ではなく、配送業務そのものは外部配送会社へ委託する。
- Hoge社は受け付けた出荷依頼に対し、自社保有在庫の引当、倉庫場所の確定、出荷可否判定、配送委託先への依頼制御を行う。
- Hoge社在庫は倉庫場所単位で管理し、保有在庫数、引当済在庫数、利用可能在庫数を区別して扱う。
- Hoge社倉庫担当者は、自身に割り当てられた倉庫場所に限り、在庫照会と入庫登録を実施できる。
- 入庫登録時は、保有在庫数と利用可能在庫数を入庫数量分だけ増加させ、引当済在庫数は変更しない。
- 引当済在庫は、取消時に引当解除し、配送受付済となった時点で出荷確定へ進める。
- Bar社は標準配送の配送受託先として実配送を行い、配送イベントを Hoge社へ通知する。
- Fuga社は特殊配送の配送受託先として実配送を行い、配送イベントを Hoge社へ通知する。

## 5. 業務連携の全体像
本資料で扱う業務連携の全体像は以下のとおりである。

1. `Foo Order Gateway` が注文確定都度、HULFTで出荷依頼ファイルを `Hoge OrderHub` へ送信する
2. `Hoge OrderHub` が受信完了をトリガに出荷依頼を取り込み、顧客確認と Hoge社保有在庫の在庫引当を行い、引当結果として倉庫場所を確定する
3. `Hoge OrderHub` が配送条件判定を行い、倉庫場所、温度帯、サイズ区分、配送先条件にもとづき、配送委託先として `Bar Delivery Center` または `Fuga Delivery Center` を選定する
4. `Hoge OrderHub` が選定結果に応じて、Bar社営業時間外は送信待ちキューへ保持した上で `Bar Delivery Center` へ配送指示を送信し、特殊配送条件に合致する場合は `Fuga Carrier Gateway` 経由で `Fuga Delivery Center` へ配送指示を送信する
5. `Bar Delivery Center` と `Fuga Delivery Center` が配送依頼受付、配送準備、配送中、配送完了などの配送イベントを時間差で `Hoge OrderHub` へ通知する
6. `Hoge OrderHub` が配送受付済イベント受信時に、引当済在庫を出荷確定へ更新する
7. `Hoge OrderHub` が配送状態を反映し、`Foo Order Gateway` へHULFTで配送状態を返却する
8. `Foo Status Client` が必要に応じて `Hoge Shipping Gateway` のAPIで出荷状態を照会する
9. `Hoge OrderHub` が `Baz` と `Qux` へ非同期連携する
10. `Hoge OrderHub` の `日次アーカイブBatch` が完了済みデータを `Hoge Sales Archive` の `S3` へ保管する
11. `Hoge社倉庫担当者` が、自身の担当倉庫場所に対する在庫照会を行い、入庫時には `Hoge Stock Keeper` へ在庫登録する
