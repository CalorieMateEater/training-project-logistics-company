# アクター定義書

## 1. 目的
本書は、要件定義工程で整理するアクターを明確化し、Hoge社の出荷・配送管理ハブに対して誰が何を期待し、どのインターフェースで関与するかを定義する。

## 2. 対象システム
- 自社対象システム: `Hoge SYS-01 Hoge OrderHub`
- 自社対象システム: `Hoge SYS-03 Hoge Stock Keeper`
- 関連システム: `Hoge SYS-05 Hoge Shipping Gateway`

| アクターID | アクター名 | 種別 | 所属 | 主目的 | 主な接点 |
| --- | --- | --- | --- | --- | --- |

## 4. アクターごとの期待事項
| アクターID | 期待事項 | Hoge側の責務 |
| --- | --- | --- |
| `ACT-002` | Foo社注文を都度受け付けたい | HULFT受信完了を契機に即時取り込み、受付通知を返却する |
| `ACT-003` | 配送中の状態を閉域APIで確認したい | 最新注文状態と配送状態を照会可能にする |
| `ACT-005` | 特殊配送案件を受け、配送イベントを順序付きで通知したい | 配送条件判定に応じてFuga社へ適切に委託し、重複送信防止情報と配送イベント順序を正しく扱う |
| `ACT-006` | 標準配送案件を冪等に受け、配送イベントを順序付きで通知したい | Bar社向け重複送信防止情報と配送イベント順序を正しく扱う |
| `ACT-007` | 請求予定を非同期で受信したい | 配送委託先への出荷依頼送信成功後に請求予定メッセージを投入する |
| `ACT-008` | 注文状態変化を非同期で受信したい | 配送委託先への出荷依頼送信成功後に注文状態通知を投入する |
| `ACT-009` | 監視しやすく、再実行・再送判断しやすい構成にしたい | ログ、監視項目、再実行条件を明文化する |
| `ACT-011` | 自分の倉庫の在庫だけを見て、入庫を即時反映したい | 担当倉庫場所単位の認可制御と、重複防止付きの入庫登録手段を提供する |

## 6. 業務時間・運用制約
| アクターID | 制約 | Hoge側の扱い |
| --- | --- | --- |
| `ACT-002` | Foo社注文は注文確定都度に HULFT 送信される | HULFT受信完了を契機に即時取り込み、受付通知返却へ接続する |
| `ACT-003` | Foo社の状態照会は 24時間365日実施され得る | 最新状態を閉域 API で常時参照可能とする |
| `ACT-005` | Fuga社は特殊配送案件を24時間365日で受け付け、配送結果通知は受付後順次返却される | 特殊配送条件に合致した案件のみ送信し、配送結果は順序制御付きで受信する |
| `ACT-006` | Bar社の出荷依頼受付は平日08:00-18:00、配送結果通知は受付後順次返却される | 出荷依頼は営業時間内送信に制御し、配送結果は順序制御付きで受信する |
| `ACT-011` | 倉庫担当者は複数名存在し、担当倉庫場所ごとに参照・登録権限を持つ | 認証済利用者に担当倉庫場所を紐づけ、他倉庫の在庫参照・入庫登録を禁止する |

## 7. 要件定義上の責任分界
| 区分 | 対象 |
| --- | --- |
| Hoge社が担う範囲 | 受付、顧客確認、自社保有在庫の在庫引当、引当解除、出荷確定、倉庫場所管理、担当倉庫場所単位の在庫照会、入庫登録、配送条件判定、Bar/Fuga委託先選定、Bar営業時間制御、配送状態管理、Foo返却、Baz/Qux通知、アーカイブ |
| Foo社が担う範囲 | 注文データ生成、HULFT送信、状態照会要求、返却ファイル取込 |
| Fuga社が担う範囲 | Hoge社から委託された特殊配送案件の実配送、配送イベント通知、配送会社内の配送資材・配送順制御 |
| Bar社が担う範囲 | Hoge社から委託された標準配送案件の実配送、配送イベント通知、配送会社内の優先制御 |

## 8. 対象外アクター
- `quux Identity Bridge`
  Foo社ネットワーク内の補助基盤であり、Hoge社との直接要件対象外とする。
- `corge Route Insight`
  Bar社ネットワーク内の分析基盤であり、Hoge社との直接要件対象外とする。
