# アクター定義書

## 1. 目的
本書は、要件定義工程で整理するアクターを明確化し、Hoge社の出荷・配送管理ハブに対して誰が何を期待し、どのインターフェースで関与するかを定義する。

## 2. 対象システム
- 自社対象システム: `Hoge SYS-01 Hoge OrderHub`
- 関連システム: `Hoge SYS-05 Hoge Shipping Gateway`

## 3. アクター一覧
| アクターID | アクター名 | 種別 | 所属 | 主目的 | 主な接点 |
| --- | --- | --- | --- | --- | --- |
| `ACT-001` | Foo社担当者 | 人 | Foo販売ソリューション株式会社 | 提携先注文の登録と配送状況確認 | Foo社システム |
| `ACT-002` | Foo Order Gateway | 外部システム | Foo販売ソリューション株式会社 | 出荷依頼ファイル送信、注文受付通知受信、配送結果受信 | ファイル連携 |
| `ACT-003` | Foo Status Client | 外部システム | Foo販売ソリューション株式会社 | 出荷状態照会 | 状態照会機能 |
| `ACT-004` | Fuga社担当者 | 人 | Fuga特殊配送サービス株式会社 | 出荷依頼登録と進捗確認 | Fuga社システム |
| `ACT-005` | Fuga API Client | 外部システム | Fuga特殊配送サービス株式会社 | 出荷依頼登録、出荷状態照会 | API連携 |
| `ACT-006` | Bar Delivery Center | 外部システム | Bar配送テクノロジー株式会社 | 出荷依頼受付、配送結果通知 | 配送依頼連携、配送結果連携 |

| `ACT-008` | Qux Mall Sync | 外部サービス | Quxモールクラウド株式会社 | 注文状態通知受信 | 非同期注文状態連携 |
| `ACT-009` | Hoge社運用担当者 | 人 | Hoge社 | Batch / Worker / API の運用監視、障害一次対応 | 監視基盤、運用手順 |
| `ACT-010` | Hoge社業務担当者 | 人 | Hoge社 | 注文・配送状態の業務確認、問い合わせ対応 | 業務報告、障害調査資料 |

## 4. アクターごとの期待事項
| アクターID | 期待事項 | Hoge側の責務 |
| --- | --- | --- |
| `ACT-002` | Foo社注文を都度受け付けたい | HULFT受信完了を契機に即時取り込み、受付通知を返却する |
| `ACT-003` | 配送中の状態を閉域APIで確認したい | 最新注文状態と配送状態を照会可能にする |
| `ACT-005` | APIで出荷依頼を登録し、送信待ちも含めて状態を確認したい | 24時間365日受付し、Bar営業時間制約を内部で吸収する |
| `ACT-006` | 冪等に出荷依頼を受け、配送イベントを順序付きで通知したい | 重複送信防止情報と配送イベント順序を正しく扱う |
| `ACT-007` | 請求予定を非同期で受信したい | Bar向け送信成功後に請求予定メッセージを投入する |
| `ACT-008` | 注文状態変化を非同期で受信したい | Bar向け送信成功後に注文状態通知を投入する |
| `ACT-009` | 監視しやすく、再実行・再送判断しやすい構成にしたい | ログ、監視項目、再実行条件を明文化する |

## 6. 業務時間・運用制約
| アクターID | 制約 | Hoge側の扱い |
| --- | --- | --- |
| `ACT-002` | Foo社注文は注文確定都度に HULFT 送信される | HULFT受信完了を契機に即時取り込み、受付通知返却へ接続する |
| `ACT-003` | Foo社の状態照会は 24時間365日実施され得る | 最新状態を閉域 API で常時参照可能とする |
| `ACT-005` | Fuga社の出荷依頼・状態照会は 24時間365日実施され得る | Bar営業時間制約は内部キューイングで吸収する |
| `ACT-006` | Bar社の出荷依頼受付は平日08:00-18:00、配送結果通知は受付後順次返却される | 出荷依頼は営業時間内送信に制御し、配送結果は順序制御付きで受信する |

## 7. 要件定義上の責任分界
| 区分 | 対象 |
| --- | --- |
| Hoge社が担う範囲 | 受付、顧客確認、在庫引当、配送条件判定、Bar営業時間制御、配送状態管理、Foo返却、Baz/Qux通知、アーカイブ |
| Foo社が担う範囲 | 注文データ生成、HULFT送信、状態照会要求、返却ファイル取込 |
| Fuga社が担う範囲 | 出荷依頼API呼出、状態照会API呼出、自社要求IDの一意性担保 |
| Bar社が担う範囲 | 実配送、配送イベント通知、配送会社内の優先制御 |

## 8. 対象外アクター
- `quux Identity Bridge`
  Foo社ネットワーク内の補助基盤であり、Hoge社との直接要件対象外とする。
- `corge Route Insight`
  Bar社ネットワーク内の分析基盤であり、Hoge社との直接要件対象外とする。
