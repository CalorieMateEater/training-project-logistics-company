# 業務機能一覧

## 1. 目的
本書は、Hoge社の出荷・配送管理業務で必要となる主要機能を一覧化し、誰に対して何を提供するのかを明確化することを目的とする。

## 2. 機能一覧
| 機能ID | 機能名 | 主アクター | 対象システム | 入力種別 | 出力種別 | 概要 |
| --- | --- | --- | --- | --- | --- | --- |
| `FNC-001` | Foo注文受付 | Foo Order Gateway | Hoge OrderHub | HULFTファイル | 注文受付結果 | Foo社からの注文を受け付ける |
| `FNC-002` | Foo注文受付通知 | Foo Order Gateway | Hoge OrderHub | なし | HULFTファイル | Hoge社で注文受付済となったことをFoo社へ通知する |
| `FNC-003` | Fuga出荷依頼受付 | Fuga API Client | Hoge Shipping Gateway / Hoge OrderHub | HTTPS API | 受付結果 | Fuga社からの出荷依頼を受け付ける |
| `FNC-004` | 顧客確認 | Hoge OrderHub | Hoge Customer Registry | 社内API | 顧客確認結果 | 注文に紐づく顧客の有効性を確認する |
| `FNC-005` | 在庫引当 | Hoge OrderHub | Hoge Stock Keeper | 社内API | 在庫引当結果 | 注文商品の在庫引当を行う |
| `FNC-006` | 配送条件判定 | Hoge OrderHub | Hoge OrderHub | 内部判断 | 配送依頼情報 | 注文元、優先度、配送条件から配送依頼内容を決定する |
| `FNC-007` | Bar社向け出荷依頼 | Bar Delivery Center | Hoge OrderHub | 内部依頼 | HTTPS API | Bar社営業時間を考慮して出荷依頼を送信する |
| `FNC-008` | 配送結果受付 | Bar Delivery Center | Hoge OrderHub | HTTPS API | 配送状態更新 | Bar社からの配送結果を受け付ける |
| `FNC-009` | Foo配送結果返却 | Foo Order Gateway | Hoge OrderHub | なし | HULFTファイル | 配送状態の変化をFoo社へ返却する |
| `FNC-010` | 出荷状態照会 | Foo Status Client / Fuga API Client | Hoge Shipping Gateway | HTTPS API | 状態照会結果 | 外部連携先から出荷状態を照会できるようにする |
| `FNC-011` | Baz請求予定通知 | Baz Billing Hub | Hoge OrderHub | なし | 非同期メッセージ | 配送依頼送信後に請求予定を通知する |
| `FNC-012` | Qux注文状態通知 | Qux Mall Sync | Hoge OrderHub | なし | 非同期メッセージ | 配送依頼送信後に注文状態を通知する |
| `FNC-013` | 日次アーカイブ | Hoge社運用担当者 | Hoge OrderHub / Hoge Sales Archive | 日次起動 | アーカイブデータ | 完了済みデータを日次で保管する |

## 3. 補足
- 本書では、機能要求としての整理を目的とし、IF-ID、物理ファイル名、URL、キュー名などの実装詳細は扱わない。
- 実装詳細は自社内部設計、および対外連携設計で定義する。
