# 業務機能一覧

## 1. 目的
本書は、Hoge社の出荷・配送管理業務で必要となる主要機能を一覧化し、誰に対して何を提供するのかを明確化することを目的とする。

## 2. 機能一覧
| 機能ID | 機能名 | 主アクター | 対象システム | 入力種別 | 出力種別 | 概要 |
| --- | --- | --- | --- | --- | --- | --- |
| `FNC-001` | Foo注文受付 | Foo Order Gateway | Hoge OrderHub | HULFTファイル | 注文受付結果 | Foo社からの注文を受け付ける |
| `FNC-002` | Foo注文受付通知 | Foo Order Gateway | Hoge OrderHub | なし | HULFTファイル | Hoge社で注文受付済となったことをFoo社へ通知する |
| `FNC-003` | Hoge直受注登録 | Hoge社業務担当者 | Hoge Shipping Gateway / Hoge OrderHub | 内部API | 受付結果 | Hoge社業務部門が直接受けた注文を登録する |
| `FNC-004` | 顧客確認 | Hoge OrderHub | Hoge Customer Registry | 社内API | 顧客確認結果 | 注文に紐づく顧客の有効性を確認する |
| `FNC-005` | 在庫引当 | Hoge OrderHub | Hoge Stock Keeper | 社内API | 在庫引当結果 | Hoge社が保有する注文商品の在庫引当を行い、倉庫場所を確定する |
| `FNC-006` | 配送条件判定 | Hoge OrderHub | Hoge OrderHub | 内部判断 | 配送依頼情報 | 注文元、優先度、倉庫場所、温度帯、サイズ区分、配送先条件から配送委託先と配送依頼内容を決定する |
| `FNC-007` | Bar社向け出荷依頼 | Bar Delivery Center | Hoge OrderHub | 内部依頼 | HTTPS API | 標準配送条件に合致する案件をBar社へ送信する |
| `FNC-008` | Fuga社向け出荷依頼 | Fuga Delivery Center | Hoge OrderHub | 内部依頼 | HTTPS API | 特殊配送条件に合致する案件をFuga社へ送信する |
| `FNC-009` | Bar配送結果受付 | Bar Delivery Center | Hoge OrderHub | HTTPS API | 配送状態更新 | 配送委託先Bar社からの配送結果を受け付ける |
| `FNC-010` | Fuga配送結果受付 | Fuga Delivery Center | Hoge OrderHub | HTTPS API | 配送状態更新 | 配送委託先Fuga社からの配送結果を受け付ける |
| `FNC-011` | Foo配送結果返却 | Foo Order Gateway | Hoge OrderHub | なし | HULFTファイル | 配送状態の変化をFoo社へ返却する |
| `FNC-012` | 出荷状態照会 | Foo Status Client | Hoge Shipping Gateway | HTTPS API | 状態照会結果 | Foo社から出荷状態を照会できるようにする |
| `FNC-013` | Baz請求予定通知 | Baz Billing Hub | Hoge OrderHub | なし | 非同期メッセージ | 配送依頼送信後に請求予定を通知する |
| `FNC-014` | Qux注文状態通知 | Qux Mall Sync | Hoge OrderHub | なし | 非同期メッセージ | 配送依頼送信後に注文状態を通知する |
| `FNC-015` | 日次アーカイブ | Hoge社運用担当者 | Hoge OrderHub / Hoge Sales Archive | 日次起動 | アーカイブデータ | 完了済みデータを日次で保管する |
| `FNC-016` | 在庫引当解除 | Hoge OrderHub | Hoge Stock Keeper | 社内API | 引当解除結果 | 未出荷取消や異常終了時に引当済在庫を解放する |
| `FNC-017` | 在庫出荷確定 | Hoge OrderHub | Hoge Stock Keeper | 社内API | 出荷確定結果 | 配送受付済となった在庫を引当済から出荷確定へ更新する |

## 3. 補足
- 本書では、機能要求としての整理を目的とし、IF-ID、物理ファイル名、URL、キュー名などの実装詳細は扱わない。
- 実装詳細は自社内部設計、および対外連携設計で定義する。
