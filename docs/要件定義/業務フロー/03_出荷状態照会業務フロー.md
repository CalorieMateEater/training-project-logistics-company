# 出荷状態照会業務フロー

## 1. 目的
Foo社からの状態照会依頼に対して、Hoge社が注文状態と配送状態を返却する業務を整理する。

## 2. 登場アクター
- 利用担当者
- 連携元システム
- Hoge社システム

## 3. 業務フロー図
```mermaid
sequenceDiagram
    box rgb(255, 244, 236) 他社
        actor User as 利用担当者
        participant Caller as 連携元システム
    end
    box rgb(237, 246, 255) 自社
        participant HogeSys as Hoge社システム
    end

    User->>Caller: 状態照会
    Caller->>HogeSys: 状態照会API呼出
    HogeSys-->>Caller: 注文状態・配送状態返却
    Caller-->>User: 照会結果表示
```

## 4. 業務の流れ
1. 利用担当者が連携元システムから状態照会を実行する。
2. 連携元システムが Hoge社システムの状態照会 API を呼び出す。
3. Hoge社システムが `partner_order_id` をもとに対象注文の注文状態、配送状態を参照する。
4. Hoge社システムが照会結果を返却する。
